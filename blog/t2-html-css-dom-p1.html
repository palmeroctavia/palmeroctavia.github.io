Can you think of an analogy to describe HTML, CSS and the DOM? How would you describe it to your non-technical friend?

I like the analogy of the body. In this model the body (website) is made up of:

  • HTML - the organs (except for skin) of a website. The content and structure are contained here;
  • CSS - the hair, eye colour and features of a website. The style of the website is contained here; and
  • DOM - the skin and bones of a website. The configuration of the organs and features are decided by how DOM places the bones and skin.

What is meant by boxifying design?

It means figuring out how the design mock up translates to the components of website structure (HTML elements e.g. text or images). Every element on a page is inside a box.

What is the box model?

The box model is the layers of an element that help position different elements on a page. The box is made up of four parts which are:

  • the element itself - the innermost layer;
  • padding - the space between the element and a border (if used);
  • the border - this can wrap an element with a border in various styles/thickness such as dashes or solid lines; and
  • the margin - the space between the border and other boxed elements.

These four parts can be added together to give the total dimensions of a boxed element.
